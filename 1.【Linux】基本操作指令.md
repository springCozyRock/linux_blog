## 5.2 mv指令（重要）

是*move*的缩写，用来移动文件或将文件改名，是Linux系统下常用命令，经常用来备份文件或目录。

```cpp
mv [选项] 源文件/目录 目标文件/目录
```

根据mv命令中第二个参数类型（是目标文件还是目标目录），mv命令将文件重命名或将其移至一个新的目录中。

- [ ] 当第二个参数是已存在的目录名称时，源文件/目录参数可以有多个，mv命令将各参数指定的源文件均移至目标目录中。
- [ ] 当第二个参数类型是文件时，mv命令完成文件重命名。此时，源文件只能有一个（也可以是源目录名），它将所给的源文件或目录重命名为给定的目标文件名。

常用选项

```c
-f  force 强制的意思，如果目标文件已经存在，不会询问而直接覆盖
-i  若目标文件. (destination) 已经存在时，就会询问是否覆盖
```



:yellow_heart: 1. 类似**剪切**功能，剪切到任意你想要的路径下 —— **目录、普通文件**

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220125105814111.png" alt="image-20220125105814111" style="zoom:80%;" />

:yellow_heart: 2. **重命名** ——**目录、普通文件

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220125105236325.png" alt="image-20220125105236325" style="zoom:80%;" />

Linux没有回收站，`rm`没有回收站就直接干掉了，下面是一段简单的shell脚本，把*rm*改造成*mv*

```c
vim ~/.bashrc #修改这个文件
mkdir -p ~/.trash
alias rm=trash
alias ur=undelfile
undelfile() {
 mv -i ~/.trash/\$@ ./
}
trash() {
 mv $@ ~/.trash/
}
```

现在不建议修改。

# 14. 打包压缩解压之类

在Linux下，打包压缩的指令很多，遇到不知道的压缩格式，复制后缀名现查现学即可。

## 14.1 zip/unzip指令

介绍它，因为它够简单，并且和windows下zip采用的是相同的压缩算法。

```c
zip file.zip file 压缩
unzip file.zip    解压
```



:yellow_heart: 1. 对**普通文件**可以**直接压缩**，且压缩效率很高

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220126120738808.png" alt="image-20220126120738808" style="zoom:80%;" />

:yellow_heart: 2. 对**目录**无法直接压缩，需要递归处理

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220126120102855.png" alt="image-20220126120102855" style="zoom:80%;" />

那怎么办呢？其实学了这么多了，就知道要递归处理

```c
zip -r dir.zip dir/
```

## 14.2 tar指令

有关tar的选项那也是非常多，不过我们重点掌握**一对儿**指令就行。

:yellow_heart: 1. **压缩**

```c
tar czf file.tgz file
```

 注：

- [ ] 后缀名`tar.gz`常简写为`tgz`
- [ ] `-`写不写取决于你的环境，我试了一下我的，写不写都行。

有关当中的选项 ——

```c
-c 	建立一个压缩文件的参数指令.(create 的意思)
-z 	是否同时具有gzip的属性？亦即是否需要用gzip压缩（没错，你可以只打包不压缩）
-f 	指明特定文件，注意！后面紧跟新文档名称！不要再加参数！
```

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220126154128416.png" alt="image-20220126154128416" style="zoom:80%;" />

有时你会看到`-v`选项，这会显示压缩过程，你怎么招都行

```c
tar czvf file.tgz file
tar cvzf file.tgz file
```

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220126154322642.png" alt="image-20220126154322642" style="zoom:67%;" />

:yellow_heart: 2. **解压缩**

这个`.tgz`文件我挪不动，所以在演示解压之前，我先把这个*lesson2*干掉，再来，

```c
tar xzf file.tgz
```

有关选项

```c
-x  解开一个压缩文件的参数指令
```

<img src="C:\Users\13136\AppData\Roaming\Typora\typora-user-images\image-20220126160034174.png" alt="image-20220126160034174" style="zoom:80%;" />



:yellow_heart: 3. **解压**到**指定路径**

```c
tar xzf file.tgz -C path
```

# 18. 关机     

注意了啊！云服务器不要关机！下面这些你想试上虚拟机上试去~

```c
shutdown [选项]
-h ： 将系统的服务停掉后，立即关机。
-r ： 在将系统的服务停掉之后就重新启动
-t sec ： -t 后面加秒数，亦即『过几秒后关机』的意思
```
